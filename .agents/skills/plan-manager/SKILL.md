# 计划管理助手（Plan Manager）

## 功能概述

本 Skill 是交通安全宣传教育工作的计划制定引擎。根据用户指定的月份，自动生成结构完整的月度宣传计划，涵盖：

1. **月度概况分析**：当月节假日、季节特征、安全风险、上级工作部署
2. **周度计划编排**：每周的宣传主题、内容方向、形式、渠道、时间节点
3. **重点内容规划**：按优先级排列的必做/建议/机动内容
4. **资源需求预估**：各类型内容的产出数量和资源配置

## 触发条件（When to Use）

当用户的输入匹配以下意图时激活本 Skill：

- 要求制定交通安全宣传计划
- 提到"制定计划""宣传计划""月度计划""排期""本月安排"
- 提到"制定X月份的宣传计划""下个月怎么安排"
- 要求规划特定时段的宣传重点
- 提到"宣传主题安排""内容排期"

**不激活**的情况：用户要求生成具体文案（由 content-producer 处理）、分析效果数据（由 analytics 处理）。

## 工作流程（Instructions）

### 步骤 1：确定计划月份

从用户输入中提取目标月份。如果用户未明确指定月份，询问或默认使用当前月份的下一个月。

需要确认的信息：

| 要素 | 说明 | 默认值 |
|------|------|--------|
| **目标月份** | YYYY年M月 | 当前月+1 |
| **特殊要求** | 用户附加的计划侧重点 | 无 |
| **参考历史** | 是否参考上月数据/计划 | 如有analytics数据则参考 |

### 步骤 2：读取计划模板

读取 `templates/plan-monthly.md` 获取月度计划的标准结构和字段定义。

### 步骤 3：分析当月特征

综合以下信息确定当月宣传重点：

#### 节假日与时间节点
根据目标月份识别：
- 国家法定节假日（春节、清明、五一、国庆等）
- 交通安全相关纪念日（全国交通安全日 12月2日等）
- 季节性时间节点（春运、暑期、开学季等）
- 地方性活动（交通安全宣传月、百日行动等）

#### 季节安全特征
根据月份确定季节性交通安全风险：
- 春季：雨雾天气、春游出行、农忙交通
- 夏季：暴雨洪水、高温疲劳、暑期出游
- 秋季：团雾多发、国庆出行、开学季
- 冬季：冰雪路面、春运高峰、年末酒驾

#### 历史数据参考
如果 `data/analytics/` 目录下有对应月份的传播数据，读取并分析：
- 上月表现最好的主题
- 各平台互动率最高的内容类型
- 需要加强的薄弱主题

如果 `data/accident-data/2025-summary.json` 可用，参考：
- 对应月份的事故高发类型
- 季节性事故特征
- 高风险区域和人群

### 步骤 4：编排周度计划

将月度分为 4-5 周，为每周安排：

| 项目 | 说明 |
|------|------|
| **周主题** | 本周宣传的核心话题 |
| **内容方向** | 2-3 个具体内容方向 |
| **内容形式** | 微信推文 / 微博 / 抖音短视频 / 海报等 |
| **发布渠道** | 微信公众号 / 微博 / 抖音 / 户外大屏等 |
| **发布时间** | 建议的发布日期和时段 |
| **配合活动** | 线下活动、联合行动等 |

编排原则：
- 主题多样化，避免连续两周重复同一主题
- 重大节假日前后安排相关安全提醒
- 结合季节特征安排应景内容
- 均衡分配各平台的内容产出

### 步骤 5：规划重点内容

按优先级分类：

- **P0 必做内容**：上级部署的主题、重大节假日安全提醒、时效性强的宣传
- **P1 建议内容**：常规安全教育、正能量宣传、知识科普
- **P2 机动内容**：根据突发热点、临时部署灵活安排

### 步骤 6：保存计划并展示

1. 按模板格式生成完整的月度计划 Markdown 文件
2. 保存到 `output/plans/{月份}-月度宣传计划-{YYYYMMDD}.md`
3. 在对话中展示完整计划
4. 提示用户可以基于计划中的主题调用 content-producer 生成具体内容

## 数据引用

| 文件路径 | 用途 |
|---------|------|
| `templates/plan-monthly.md` | 月度计划结构模板 |
| `data/analytics/wechat-202601.json` | 微信传播数据（分析参考） |
| `data/analytics/weibo-202601.json` | 微博传播数据（分析参考） |
| `data/analytics/douyin-202601.json` | 抖音传播数据（分析参考） |
| `data/accident-data/2025-summary.json` | 事故数据（风险参考） |
| `output/plans/` | 计划输出目录 |

## 输出格式

计划输出为 Markdown 文件，严格遵循 `templates/plan-monthly.md` 的结构，包含以下核心章节：

1. **基本信息**：计划月份、制定日期、制定人、审批状态
2. **本月概况**：节假日时间节点、季节安全风险、上级工作部署
3. **周度计划**：4-5周，每周含主题/内容方向/形式/渠道/时间/活动
4. **重点内容规划**：P0必做 / P1建议 / P2机动
5. **资源需求**：各类型内容数量
6. **效果预期**：各平台的目标指标

## 约束规则

### 计划编排规则
- 每月至少安排 4 个不同的周主题
- 每周至少安排 2 种内容形式（如微信推文 + 抖音视频）
- 重大节假日（如春节、国庆）必须提前一周安排相关主题
- 月度计划中必须包含至少 1 个交警正能量主题
- 月度计划中必须包含至少 1 个法制宣传主题

### 内容数量规则
- 月度微信推文建议 12-16 篇
- 月度微博发布建议 20-30 条
- 月度抖音视频建议 8-12 条
- 海报设计建议 4-8 张

### 时效性规则
- 时令性内容（如春运、高考、暑期）必须在对应时段安排
- 天气相关宣传跟随季节自动安排（冬季冰雪、夏季暴雨等）
- 结合近期热点事件灵活调整（由用户提供热点信息）

### 文件命名
- 格式：`{月份}-月度宣传计划-{YYYYMMDD}.md`
- 示例：`3月-月度宣传计划-20260225.md`

## 使用示例

### 示例 1：生成月度计划

```
用户: 帮我制定3月份的宣传计划

Agent 执行流程:
1. 确定目标月份: 2026年3月
2. 读取计划模板
3. 分析3月特征: 春季开学、全国两会、春雨增多、春游出行
4. 参考数据: 读取历史传播数据和事故数据
5. 编排4周计划
6. 生成完整月度计划文档
7. 保存到 output/plans/
```

### 示例 2：带特殊要求的计划

```
用户: 制定2月份的宣传计划，重点要突出春运安全和节后返工

Agent 执行流程:
1. 确定目标月份: 2026年2月
2. 重点标记: 春运安全、节后返工
3. 分析2月特征: 春节、元宵、春运返程高峰
4. 在周度安排中优先安排用户指定的重点主题
5. 生成计划
```

### 示例 3：计划驱动内容生产

```
用户: 根据计划里第一周的主题，帮我生成相关文案
Agent: [读取计划 -> 提取第一周主题 -> 调用 content-producer 生成文案]
```

### 示例 4：参考上月数据优化计划

```
用户: 制定2月份的计划，参考1月份的传播效果数据

Agent 执行流程:
1. 读取 data/analytics/ 下的1月数据
2. 分析各主题表现，识别高互动主题和低互动主题
3. 在2月计划中增加高互动主题的比重，优化低互动主题的策略
4. 生成计划
```
