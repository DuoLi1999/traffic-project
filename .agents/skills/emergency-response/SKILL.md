# 应急宣传响应助手（Emergency Response）

## 功能概述

本 Skill 在收到预警信息或突发事件通知时，快速生成一套完整的应急宣传内容包。应急内容包包含：

1. **安全提示**：面向公众的安全行车/出行提示（微信、微博、抖音三平台版本）
2. **预警通报**：正式的预警信息通报（适用于官方发布）
3. **绕行建议**：受影响路段的绕行方案和出行建议
4. **应急联系**：报警、救援、路政等应急联系方式汇总

设计目标是在 **最短时间内** 产出可发布的多平台应急内容，确保预警信息及时传达公众。

## 触发条件（When to Use）

当用户的输入匹配以下意图时激活本 Skill：

- 收到恶劣天气预警，需要发布应急宣传
- 提到"暴雨预警""冰雪预警""大雾预警""台风预警"等天气预警
- 发生重大交通事故或交通管制，需要紧急发布信息
- 提到"应急宣传""应急响应""紧急发布""预警发布"
- 提到"道路封闭""交通管制""高速封路"需要发布绕行信息
- 提到"发布预警""启动应急宣传"

**不激活**的情况：常规宣传内容生成（由 content-producer 处理）、计划性工作安排（由 plan-manager 处理）。

## 工作流程（Instructions）

### 步骤 1：收集预警信息

从用户输入中提取应急事件的关键信息：

| 要素 | 说明 | 是否必填 |
|------|------|---------|
| **事件类型** | 天气预警/交通事故/道路施工/交通管制/其他 | 必填 |
| **预警等级** | 蓝色/黄色/橙色/红色（天气预警适用） | 天气预警必填 |
| **影响范围** | 受影响的路段、区域 | 必填 |
| **影响时间** | 预计影响的起止时间 | 建议提供 |
| **具体情况** | 事件的详细描述 | 建议提供 |
| **管制措施** | 已采取或将采取的管制措施 | 如有则填 |

如果用户提供的信息不完整，先基于已有信息生成内容，同时标注信息待补充的位置。

### 步骤 2：确定内容包结构

根据事件类型和严重程度，确定需要生成的内容：

| 预警等级 | 安全提示 | 预警通报 | 绕行建议 | 应急联系 |
|---------|---------|---------|---------|---------|
| 蓝色（一般） | 必出 | 可选 | 如涉及道路则出 | 可选 |
| 黄色（较重） | 必出 | 必出 | 必出 | 必出 |
| 橙色（严重） | 必出 | 必出 | 必出 | 必出 |
| 红色（特别严重） | 必出 | 必出 | 必出 | 必出 |

### 步骤 3：生成安全提示（三平台版本）

针对微信、微博、抖音分别生成安全提示内容：

#### 微信版安全提示
- 字数：500-800 字（比常规推文短，强调时效性）
- 结构：预警信息 -> 安全要点（3-5条）-> 绕行建议 -> 应急联系
- 风格：正式但紧迫，信息优先
- 参考模板：`templates/content-wechat.md`（简化版）

#### 微博版安全提示
- 字数：140 字以内
- 结构：`#话题标签#` + 预警信息 + 核心安全提示 + `#话题标签#`
- 风格：简洁明了、信息密度高
- 参考模板：`templates/content-weibo.md`

#### 抖音版安全提示
- 字数：200 字以内
- 结构：紧迫感 Hook -> 核心安全要点 -> 行动呼吁
- 风格：口语化但严肃
- 参考模板：`templates/content-douyin.md`

### 步骤 4：生成预警通报

正式的预警通报格式：

```markdown
# 交通安全预警通报

## 预警信息
- **预警类型**: {类型}
- **预警等级**: {蓝色/黄色/橙色/红色}
- **发布时间**: {YYYY-MM-DD HH:mm}
- **影响范围**: {具体路段/区域}
- **预计持续**: {时间段}

## 情况说明
{事件详细描述}

## 管制措施
{已采取的交通管制措施}

## 安全提示
1. {提示1}
2. {提示2}
3. {提示3}

## 绕行方案
{详细的绕行路线建议}

## 应急联系
- 报警电话：110
- 交通事故报警：122
- 高速公路报警救援：12122
- 道路信息查询：{电话}
```

### 步骤 5：生成绕行建议

基于受影响路段，生成绕行方案：
- 列出受影响的具体路段
- 提供 2-3 条替代路线
- 估计绕行增加的时间
- 标注替代路线的路况提醒

### 步骤 6：打包保存并展示

1. 将所有内容打包保存到 `output/content/` 目录：
   - `应急-{事件关键词}-微信-{YYYYMMDD}.md`
   - `应急-{事件关键词}-微博-{YYYYMMDD}.md`
   - `应急-{事件关键词}-抖音-{YYYYMMDD}.md`
   - `应急-{事件关键词}-通报-{YYYYMMDD}.md`
2. 在对话中依次展示所有生成的内容
3. 标注内容已就绪，可直接发布
4. 提醒用户确认关键信息（路段、时间、管制措施）的准确性

## 约束规则

### 时效性要求
- 应急内容必须在最短时间内生成，优先保证核心信息完整
- 信息不完整时先输出已有内容，标注 `[待确认: ...]` 需要补充的信息
- 简化审核流程：应急内容标注"应急发布，请人工快速确认后发布"

### 内容规范
- 预警等级必须与官方发布一致，不得自行升降级
- 路段信息必须具体准确（精确到路名、路段编号、公里桩号）
- 绕行方案必须可行，不建议明显不合理的路线
- 安全提示必须实用、可操作，避免空洞的套话

### 语言风格
- 正式但不死板，兼顾紧迫感和可读性
- 核心信息（路段、时间、措施）用加粗或高亮强调
- 避免引起过度恐慌的表述（如"极其危险""绝对不要出门"）
- 使用"建议""请""注意"等引导性用语

### 信息安全
- 不发布未经确认的事故伤亡信息
- 不公开具体的救援部署细节
- 不发布可能影响救援工作的信息

### 文件命名
- 格式：`应急-{事件关键词}-{平台/通报}-{YYYYMMDD}.md`
- 示例：`应急-暴雨橙色预警-微信-20260211.md`

## 使用示例

### 示例 1：天气预警应急

```
用户: 发布暴雨橙色预警应急宣传，影响G15和G50高速

Agent 执行流程:
1. 提取: 事件=暴雨, 等级=橙色, 影响范围=G15和G50高速
2. 确定内容包: 安全提示x3 + 预警通报 + 绕行建议 + 应急联系
3. 生成微信/微博/抖音三版安全提示
4. 生成预警通报（含绕行方案和应急联系）
5. 打包保存4个文件到 output/content/
6. 展示全部内容
```

### 示例 2：交通管制应急

```
用户: 城区人民路因施工临时封闭，需要发布绕行通知

Agent 执行流程:
1. 提取: 事件=道路施工封闭, 影响范围=人民路
2. 确定内容包: 安全提示x3 + 绕行建议
3. 生成三平台通知内容（含绕行方案）
4. 保存并展示
```

### 示例 3：冰雪天气预警

```
用户: 明天有寒潮来袭，预计山区道路结冰，启动应急宣传

Agent 执行流程:
1. 提取: 事件=寒潮/道路结冰, 影响范围=山区道路
2. 参考事故数据中的冰雪路面风险信息
3. 生成冰雪天气安全驾驶提示内容包
4. 保存并展示
```
