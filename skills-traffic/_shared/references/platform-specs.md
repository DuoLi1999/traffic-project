# 三平台格式规范

> 微信公众号、微博、抖音三个平台的内容格式规范汇总。

## 微信公众号推文

| 项目 | 规范 |
|------|------|
| **字数** | 800 - 1500 字 |
| **结构** | 标题 → 导语 → 正文（3-4个小标题分段）→ 结尾呼吁 → 编辑信息 |
| **标题** | 吸引眼球但不标题党，可用数字、疑问句、感叹句 |
| **正文** | 信息密度高，兼顾可读性，每段 3-5 行 |
| **配图** | 在合适位置标注 `[配图建议: 描述]` |
| **尾部** | 固定格式：来源 / 编辑 / 审核 |
| **emoji** | 可适量使用辅助排版，不宜过多 |

### 应急模式
- 字数缩短为 500-800 字
- 强调时效性，信息优先
- 结构：预警信息 → 安全要点（3-5条）→ 绕行建议 → 应急联系

## 微博短文

| 项目 | 规范 |
|------|------|
| **字数** | 核心信息 ≤ 140 字（不含话题标签） |
| **结构** | `#话题标签#` + 核心信息 + 行动呼吁 + `#话题标签#` |
| **话题标签** | 前后各 1-2 个，格式 `#话题名#` |
| **风格** | 简洁有力、口语化、适度使用 emoji、善用数字和对比 |
| **配图** | 标注配图数量和类型建议 |

## 抖音短视频文案

| 项目 | 规范 |
|------|------|
| **字数** | ≤ 300 字 |
| **结构** | Hook（前3秒抓眼球）→ 核心内容（2-4句）→ 行动呼吁 |
| **风格** | 口语化、节奏感强、接地气、有"梗"或反转 |
| **话题标签** | 3-5 个，放在文案末尾 |
| **开头** | 必须有"钩子"（疑问/反转/冲突），让用户停下来看 |
| **语气** | 像朋友聊天，避免公文腔，可适当用网络流行语 |

### 应急模式
- 字数缩短为 ≤ 200 字
- 风格口语化但严肃
- 结构：紧迫感 Hook → 核心安全要点 → 行动呼吁

## 通用规则

### 文件命名
- 常规内容：`{主题关键词}-{平台}-{YYYYMMDD}.md`
- 应急内容：`应急-{事件关键词}-{平台/通报}-{YYYYMMDD}.md`
- 海报方案：`{主题关键词}-海报方案-{YYYYMMDD}.json`
- 视频脚本：`{主题关键词}-脚本-{YYYYMMDD}.md`

### 输出目录
- 文案 → `output/content/`
- 海报方案 → `output/posters/`
- 视频脚本 → `output/scripts/`
- 分析报告 → `output/reports/`
- 月度计划 → `output/plans/`
