# 计划管理助手（Plan Manager）

## 功能概述

本 Skill 是交通安全宣传教育工作的计划制定引擎。根据用户指定的月份，自动生成结构完整的月度宣传计划，涵盖：

1. **月度概况分析**：当月节假日、季节特征、安全风险、上级工作部署
2. **周度计划编排**：每周的宣传主题、内容方向、形式、渠道、时间节点
3. **重点内容规划**：按优先级排列的必做/建议/机动内容
4. **资源需求预估**：各类型内容的产出数量和资源配置

## 触发条件

当用户消息包含以下关键词时激活：
- 制定计划、宣传计划、月度计划、排期、本月安排
- 制定X月份的宣传计划、下个月怎么安排
- 宣传主题安排、内容排期

**不激活**的情况：生成具体文案（content-producer）、分析效果数据（analytics）。

## 工作流程

### 1. 确认计划月份
- 从用户输入提取目标月份（默认当前月+1）
- 确认特殊要求和侧重点
- 确认是否参考历史数据

### 2. 读取计划模板
- 读取 `assets/templates/plan-monthly.md` 获取月度计划标准结构

### 3. 分析当月特征
- 识别当月节假日与时间节点（参考 `references/calendar-reference.md`）
- 确定季节性交通安全风险
- 读取 `_shared/assets/data/analytics/` 下的历史传播数据（如有）
- 读取 `_shared/assets/data/accident-data/2025-summary.json` 参考事故特征（如有）

### 4. 编排周度计划
- 将月度分为 4-5 周，每周安排主题、内容方向、形式、渠道、时间、活动
- 主题多样化，节假日提前安排，均衡各平台产出

### 5. 规划重点内容
- 按 P0必做 / P1建议 / P2机动 分级排列内容优先级

### 6. 保存计划
- 按模板格式生成 Markdown 文件
- 保存到 `output/plans/{月份}-月度宣传计划-{YYYYMMDD}.md`
- 展示计划并提示可调用 content-producer 生成具体内容

## 数据引用

| 文件路径 | 用途 |
|---------|------|
| `assets/templates/plan-monthly.md` | 月度计划结构模板 |
| `references/calendar-reference.md` | 节假日与交通安全日历 |
| `_shared/assets/data/analytics/` | 各平台传播数据（分析参考） |
| `_shared/assets/data/accident-data/2025-summary.json` | 事故数据（风险参考） |
| `output/plans/` | 计划输出目录 |

## 输出格式

计划输出为 Markdown 文件，严格遵循 `assets/templates/plan-monthly.md` 的结构：

1. **基本信息**：计划月份、制定日期、制定人、审批状态
2. **本月概况**：节假日时间节点、季节安全风险、上级工作部署
3. **周度计划**：4-5周，每周含主题/内容方向/形式/渠道/时间/活动
4. **重点内容规划**：P0必做 / P1建议 / P2机动
5. **资源需求**：各类型内容数量
6. **效果预期**：各平台的目标指标

## 约束规则

### 计划编排规则
- 每月至少 4 个不同周主题，每周至少 2 种内容形式
- 重大节假日必须提前一周安排相关主题
- 必须包含至少 1 个交警正能量主题和 1 个法制宣传主题

### 内容数量规则
- 微信推文 12-16 篇/月，微博 20-30 条/月
- 抖音视频 8-12 条/月，海报 4-8 张/月

### 时效性规则
- 时令性内容必须在对应时段安排
- 天气相关宣传跟随季节自动安排
- 结合近期热点灵活调整

### 文件命名
- 格式：`{月份}-月度宣传计划-{YYYYMMDD}.md`
- 示例：`3月-月度宣传计划-20260225.md`

> 详细规则和使用示例请参考 `references/planning-rules.md` 和 `references/examples.md`。
