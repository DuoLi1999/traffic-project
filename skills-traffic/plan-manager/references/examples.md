# 使用示例

## 示例 1：生成月度计划

```
用户: 帮我制定3月份的宣传计划

Agent 执行流程:
1. 确定目标月份: 2026年3月
2. 读取计划模板 assets/templates/plan-monthly.md
3. 分析3月特征: 春季开学、全国两会、春雨增多、春游出行
4. 参考数据: 读取历史传播数据和事故数据
5. 编排4周计划
6. 生成完整月度计划文档
7. 保存到 output/plans/
```

## 示例 2：带特殊要求的计划

```
用户: 制定2月份的宣传计划，重点要突出春运安全和节后返工

Agent 执行流程:
1. 确定目标月份: 2026年2月
2. 重点标记: 春运安全、节后返工
3. 分析2月特征: 春节、元宵、春运返程高峰
4. 在周度安排中优先安排用户指定的重点主题
5. 生成计划
```

## 示例 3：计划驱动内容生产

```
用户: 根据计划里第一周的主题，帮我生成相关文案
Agent: [读取计划 -> 提取第一周主题 -> 调用 content-producer 生成文案]
```

## 示例 4：参考上月数据优化计划

```
用户: 制定2月份的计划，参考1月份的传播效果数据

Agent 执行流程:
1. 读取 _shared/assets/data/analytics/ 下的1月数据
2. 分析各主题表现，识别高互动主题和低互动主题
3. 在2月计划中增加高互动主题的比重，优化低互动主题的策略
4. 生成计划
```

## 输出格式说明

计划输出为 Markdown 文件，严格遵循 `assets/templates/plan-monthly.md` 的结构，包含以下核心章节：

1. **基本信息**：计划月份、制定日期、制定人、审批状态
2. **本月概况**：节假日时间节点、季节安全风险、上级工作部署
3. **周度计划**：4-5周，每周含主题/内容方向/形式/渠道/时间/活动
4. **重点内容规划**：P0必做 / P1建议 / P2机动
5. **资源需求**：各类型内容数量
6. **效果预期**：各平台的目标指标

### 周度计划表格结构

每周的计划使用以下表格格式：

| 项目 | 内容 |
|------|------|
| **周主题** | 本周宣传的核心话题 |
| **内容方向** | 2-3 个具体内容方向 |
| **内容形式** | 微信推文/微博/抖音短视频/海报等 |
| **发布渠道** | 微信公众号/微博/抖音/户外大屏等 |
| **发布时间** | 建议的发布日期和时段 |
| **配合活动** | 线下活动、联合行动等 |
