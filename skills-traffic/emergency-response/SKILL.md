# emergency-response — 交通安全应急宣传响应助手

## 功能概述

根据预警信息快速生成应急宣传内容包，支持以下输出：

| 输出内容 | 说明 |
|----------|------|
| **安全提示（三平台）** | 分别为微信公众号、微博、抖音生成符合各平台应急模式格式的安全提示 |
| **预警通报** | 结构化预警通报文档，含情况说明、管制措施、安全提示 |
| **绕行建议** | 受影响路段的替代路线方案与出行建议 |
| **应急联系** | 附带标准应急联系电话清单 |

## 触发条件

当用户消息包含以下关键词或意图时激活：
- 天气类：暴雨、冰雪、大雾、预警
- 事件类：突发、应急宣传、应急响应、紧急发布
- 组合意图：用户描述恶劣天气或突发事件并要求生成宣传内容

## 工作流程

### 1. 收集预警信息

从用户输入中提取以下关键要素：
- **事件类型**：暴雨 / 冰雪 / 大雾 / 交通事故 / 道路施工 / 其他
- **预警等级**：蓝色 / 黄色 / 橙色 / 红色
- **影响区域**：具体路段、区域
- **持续时间**：预计起止时间
- **详细情况**：事件描述、已知影响
- **已采取措施**：交通管制、应急处置等

信息不全时主动向用户确认补充。

### 2. 确定内容包范围

根据预警等级确定输出内容，详见 `references/alert-levels.md`：

| 预警等级 | 安全提示 | 预警通报 | 绕行建议 |
|----------|----------|----------|----------|
| 蓝色（一般） | 必须 | 可选 | 涉及路段时提供 |
| 黄色（较重） | 必须 | 必须 | 必须 |
| 橙色（严重） | 必须 | 必须 | 必须 |
| 红色（特别严重） | 必须 | 必须 | 必须 |

### 3. 生成安全提示（三平台）

按照 `_shared/references/platform-specs.md` 中各平台**应急模式**规范，分别生成：

- **微信公众号推文**（500-800字）：预警信息 → 安全要点（3-5条）→ 绕行建议 → 应急联系
- **微博短文**（≤140字）：#话题标签# + 核心安全信息 + 行动呼吁 + #话题标签#
- **抖音文案**（≤200字）：紧迫感Hook → 核心安全要点 → 行动呼吁

### 4. 生成预警通报

使用 `assets/templates/alert-notice-template.md` 模板，填充以下内容：
- 预警信息（类型、等级、时间、范围）
- 情况说明、管制措施
- 安全提示和绕行方案

### 5. 生成绕行建议

使用 `assets/templates/detour-template.md` 模板，提供：
- 受影响路段清单
- 至少两条替代路线方案（含预计增加时间和路况提醒）
- 综合出行建议

### 6. 打包输出

将所有生成内容保存至 `output/content/` 目录，文件命名遵循应急格式：
`应急-{事件关键词}-{平台/通报/绕行}-{YYYYMMDD}.md`

## 约束规则

1. **时效性优先**：应急内容以速度为第一要求，信息准确、结构清晰、用语简练，避免冗长修饰。
2. **准确性要求**：所有路段名称、管制措施、绕行路线必须基于用户提供的信息，不可编造具体地名和路线。
3. **语气把控**：正式但不生硬，传递紧迫感但不制造恐慌。避免使用"可能""也许"等模糊表述，使用明确指令性语言。
4. **信息安全**：不泄露内部处置细节、伤亡具体数字（除非用户明确要求公开）、涉密信息。遵守 `_shared/references/content-redlines.md` 中的合规要求。
5. **联系方式**：必须附带标准应急联系电话，具体内容见 `references/emergency-contacts.md`。
6. **平台规范**：三平台安全提示必须遵循各平台应急模式格式要求，详见 `_shared/references/platform-specs.md`。
