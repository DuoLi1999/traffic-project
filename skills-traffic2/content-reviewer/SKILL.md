# 内容审核助手（Content Reviewer）

## 功能概述

本 Skill 是交通安全宣传教育内容的质量把关系统，对宣传内容进行三级审核：

1. **初审（格式与敏感词）**：检查错别字、敏感词、格式规范
2. **复审（内容质量）**：评估内容逻辑、案例真实性、表达准确性
3. **终审（政策合规）**：确保符合宣传政策、法律法规、保密要求

输出详细的审核报告，标记问题并给出修改建议。

## 触发条件（When to Use）

当用户的输入匹配以下意图时激活本 Skill：

- 要求审核宣传内容
- 提到"审核""检查""审一下""看看有没有问题"
- 提到"合规检查""政策审查"
- 提到"帮我看看这篇文案"
- 提交内容要求"把关""确认"

**不激活**的情况：用户要求生成内容（由 content-producer 处理）、制定计划（由 plan-manager 处理）。

## 工作流程（Instructions）

### 步骤 1：确定审核范围

确认需要审核的内容和审核重点：

| 要素 | 说明 | 询问/确认 |
|------|------|----------|
| **内容类型** | 推文/微博/视频脚本/海报 | 必须确认 |
| **发布平台** | 微信/微博/抖音等 | 必须确认 |
| **审核级别** | 初审/复审/终审/全套 | 默认全套 |
| **特殊要求** | 用户关注的特定问题 | 可选 |

### 步骤 2：加载审核标准

读取审核资源文件：
- `reference/sensitive-words.json` - 敏感词库
- `reference/policy-checklist.md` - 政策合规检查清单
- `reference/quality-standards.md` - 内容质量标准

### 步骤 3：执行三级审核

#### 初审：格式与敏感词检查

**检查项：**
- [ ] 错别字检查
- [ ] 敏感词检测（政治、血腥、歧视等）
- [ ] 格式规范（标点、空格、段落）
- [ ] 平台字数限制
- [ ] 标题规范

**输出：** 初审问题清单

#### 复审：内容质量检查

**检查项：**
- [ ] 逻辑结构是否清晰
- [ ] 案例是否真实可信
- [ ] 数据是否准确（来源是否可靠）
- [ ] 表述是否准确（无误导性）
- [ ] 目标受众是否明确
- [ ] 行动号召是否清晰
- [ ] 可读性评估

**输出：** 复审问题清单

#### 终审：政策合规检查

**检查项：**
- [ ] 是否符合交通安全宣传政策
- [ ] 是否涉及保密信息（具体案件细节）
- [ ] 隐私保护（个人信息打码）
- [ ] 画面内容适度性（血腥度）
- [ ] 法律条款引用准确性
- [ ] 是否可能引发舆情风险
- [ ] 正能量导向

**输出：** 终审问题清单

### 步骤 4：生成审核报告

整合三级审核结果，生成结构化报告：

```markdown
# 内容审核报告

## 基本信息
- 审核内容：{标题/主题}
- 内容类型：{类型}
- 发布平台：{平台}
- 审核时间：{时间}
- 审核员：Content Reviewer AI

## 审核结论
[ ] 通过 - 可以直接发布
[ ] 有条件通过 - 修改后发布
[ ] 不通过 - 需要重大修改

## 初审结果（格式与敏感词）
状态：{通过/问题数}

### 发现的问题
1. **{问题类型}**：{具体描述}
   - 位置：{章节/段落}
   - 原文："{原文内容}"
   - 建议：{修改建议}

2. ...

## 复审结果（内容质量）
状态：{通过/问题数}

### 发现的问题
...

## 终审结果（政策合规）
状态：{通过/问题数}

### 发现的问题
...

## 综合建议

### 必须修改（不修改不能发布）
1. {修改项1}
2. {修改项2}

### 建议修改（优化内容质量）
1. {修改项1}
2. {修改项2}

### 可选优化
1. {优化项1}

## 修改后复查
修改完成后，请重新提交审核。
```

### 步骤 5：输出与保存

1. 在对话中展示审核报告
2. 保存到 `output/review/{日期}-{内容标题}-审核报告.md`
3. 如发现问题严重，提醒用户修改后重新提交

## 审核标准

### 敏感词分类

#### 政治敏感
- 涉及国家领导人的不当表述
- 敏感政治事件
- 不当政治言论

#### 血腥暴力
- 过度血腥描述
- 恐怖细节渲染
- 残忍行为描述

#### 歧视性语言
- 地域歧视
- 职业歧视
- 性别歧视

#### 不当表述
- 对交警部门的不当言论
- 对当事人的恶意描述
- 可能引发舆情的表述

### 内容质量标准

#### 逻辑性
- 结构清晰，层次分明
- 论点论据匹配
- 因果逻辑合理

#### 准确性
- 事实准确无误
- 数据真实可查
- 法律条款正确

#### 可读性
- 语言通顺流畅
- 通俗易懂
- 避免专业术语堆砌

### 政策合规标准

#### 保密要求
- 不披露案件侦查细节
- 不泄露个人隐私
- 不公布涉密信息

#### 适度原则
- 画面处理适度（不过度血腥）
- 警示与教育并重
- 避免引发恐慌

#### 正能量导向
- 弘扬正能量
- 体现交警正面形象
- 传递安全理念

## 问题等级定义

| 等级 | 定义 | 处理方式 |
|------|------|---------|
| **致命** | 严重违反政策/法律/安全 | 必须修改，否则不能发布 |
| **严重** | 明显错误，影响内容质量 | 建议修改，影响发布 |
| **一般** | 小问题，可优化 | 可选修改，不阻止发布 |
| **提示** | 优化建议 | 仅供参考 |

## 约束规则

### 审核原则
- 严格把关，不降低标准
- 客观公正，有据可查
- 明确问题，给出建议
- 保护隐私，注意保密

### 不予通过的情形
- 包含敏感政治内容
- 过度血腥暴力
- 泄露案件秘密
- 可能引发负面舆情
- 事实严重错误
- 法律条款错误引用

### 审核记录
- 保存所有审核报告
- 记录审核时间和内容版本
- 跟踪修改情况

## 使用示例

### 示例 1：常规审核

```
用户: 帮我审核一下这篇推文

Agent 执行流程:
1. 确认内容类型：微信公众号推文
2. 加载审核标准
3. 执行三级审核：
   - 初审：检查错别字、敏感词
   - 复审：评估内容质量
   - 终审：政策合规检查
4. 生成审核报告
5. 指出2个一般问题，1个建议修改
6. 保存审核报告
```

### 示例 2：快速检查

```
用户: 快速检查一下这条微博有没有问题

Agent 执行流程:
1. 执行初审（格式+敏感词）
2. 快速确认无明显问题
3. 提示"初审通过，建议发布前再次确认"
```

### 示例 3：问题内容审核

```
用户: 审核一下这篇关于事故的报道

Agent 执行流程:
1. 发现内容包含未打码的车牌号（隐私问题）
2. 发现画面血腥度较高（适度问题）
3. 标注为"有条件通过"
4. 明确要求：
   - 必须打码处理车牌
   - 建议模糊血腥画面
5. 生成详细审核报告
```

### 示例 4：法规引用检查

```
用户: 检查一下这段法规引用是否正确

Agent 执行流程:
1. 核对法规条款（参考 legal-terms.md）
2. 发现引用条款已过期
3. 提供最新条款原文
4. 建议修改引用
```
