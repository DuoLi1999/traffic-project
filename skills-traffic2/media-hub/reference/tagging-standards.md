# 标签标引标准

## 一、标签体系架构

### 标签分类层级

```
素材标签体系
├── 事故维度
│   ├── 事故类型
│   ├── 事故严重程度
│   └── 事故形态
├── 违法维度
│   ├── 违法类型
│   ├── 违法主体
│   └── 违法情节
├── 环境维度
│   ├── 天气条件
│   ├── 道路类型
│   ├── 时间段
│   └── 地理位置
├── 对象维度
│   ├── 车辆类型
│   ├── 人员类型
│   └── 涉及关系
└── 用途维度
    ├── 宣传类型
    ├── 教育目的
    └── 适用场景
```

## 二、标签命名规范

### 命名原则

1. **简洁性**：2-4个汉字
2. **准确性**：准确描述素材特征
3. **一致性**：同一概念统一用词
4. **专业性**：使用行业标准术语

### 命名格式

```
[维度]-[具体描述]

示例：
事故-追尾
天气-暴雨
道路-高速
```

实际存储时省略维度前缀，直接存储具体标签。

## 四、标签组合规则

### 必选项（每条素材至少包含）
1. 事故类型 或 违法类型（二选一或都选）
2. 道路类型
3. 车辆类型 或 人员类型（至少一个）
4. 宣传类型

### 可选项（根据内容添加）
- 天气条件（如涉及）
- 时间特征（如涉及）
- 其他描述性标签

### 标签数量
- 最少：3个标签
- 建议：5-6个标签
- 最多：8个标签

## 五、标引示例

### 示例 1：雨天追尾事故视频

**素材描述**：
2026年2月10日，雨天，高速公路，白色小轿车追尾前方货车，两车受损，无人员伤亡。

**生成标签**：
```json
{
  "tags": [
    "追尾",           // 事故类型-必选
    "雨天",           // 天气条件
    "高速公路",       // 道路类型-必选
    "小型客车",       // 车辆类型-必选
    "货车",           // 涉及车辆
    "警示教育"        // 宣传类型-必选
  ]
}
```

交警夜间设卡检查，查获一名涉嫌酒驾的司机，现场进行酒精测试。

```json
}
```

## 六、质量标准

### 准确性
- [ ] 标签准确描述素材内容
- [ ] 无错误标签
- [ ] 无无关标签

### 完整性
- [ ] 包含必选项
- [ ] 覆盖主要特征
- [ ] 不遗漏重要信息

### 一致性
- [ ] 使用标准标签词
- [ ] 同类素材标签一致
- [ ] 无重复标签

### 规范性
- [ ] 符合命名规范
- [ ] 数量在合理范围
- [ ] 标签顺序合理

## 八、自动化标引建议

### 可从文件名提取的信息
- 日期：`20260210` → 2026年2月10日
- 地点：`highway` → 高速公路
- 类型：`accident` → 事故

### 可从描述提取的关键词
- 使用 NLP 技术提取实体
- 识别时间、地点、人物、事件
- 匹配标签词典

### 人工审核点
- 标签准确性
- 完整性检查
- 特殊标签补充
