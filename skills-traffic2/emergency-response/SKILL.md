# 应急响应助手（Emergency Response）

## 功能概述

本 Skill 是交通安全宣传教育工作的应急响应系统，针对恶劣天气和突发事件快速生成应急宣传内容：

1. **天气预警响应**：暴雨、冰雪、大雾等恶劣天气预警
2. **突发事件响应**：交通事故、道路中断、重大活动等
3. **快速内容生成**：30分钟内生成应急内容包
4. **多渠道适配**：一键生成多平台应急内容
5. **时效管理**：追踪应急内容发布和效果

## 触发条件（When to Use）

当用户的输入匹配以下意图时激活本 Skill：

- 发布应急宣传内容
- 提到"应急""预警""暴雨""冰雪""大雾"
- 提到"突发事件""紧急发布""路况提醒"
- 提到"恶劣天气""道路管制"
- 收到气象/应急部门预警信息

**不激活**的情况：常规宣传计划制定、日常内容生成。

## 工作流程（Instructions）

### 步骤 1：识别应急类型

确认应急事件类型和级别：

| 类型 | 子类型 | 级别 | 说明 |
|------|--------|------|------|
| **天气预警** | 暴雨、大雾、冰雪、台风、高温 | 蓝/黄/橙/红 | 气象部门发布 |
| **道路事件** | 事故、施工、管制、中断 | 一般/严重 | 交警部门通报 |
| **突发事件** | 自然灾害、公共安全 | 一般/重大 | 上级部门通报 |
| **重大活动** | 赛事、会议、节假日 | - | 提前规划 |

### 步骤 2：读取应急模板

根据应急类型读取对应模板：

**天气类**：
- `reference/emergency-templates/rainstorm.md` - 暴雨模板
- `reference/emergency-templates/snow-ice.md` - 冰雪模板
- `reference/emergency-templates/fog-warning.md` - 大雾模板
- `reference/emergency-templates/typhoon.md` - 台风模板
- `reference/emergency-templates/heat-wave.md` - 高温模板

**事件类**：
- `reference/emergency-templates/accident.md` - 事故模板
- `reference/emergency-templates/road-closure.md` - 道路管制模板

### 步骤 3：收集关键信息

根据应急类型收集必要信息：

#### 天气预警信息
- 预警类型（暴雨/大雾/冰雪等）
- 预警级别（蓝/黄/橙/红）
- 影响时间（开始-结束）
- 影响区域
- 预计影响程度

#### 道路事件信息
- 事件类型
- 发生地点
- 影响路段
- 预计持续时间
- 管制措施
- 绕行建议

### 步骤 4：生成应急内容包

根据模板和信息，生成完整内容包：

#### 内容包结构

```
emergency-content-{时间戳}/
├── wechat-urgent.md          # 微信公众号推文
├── weibo-urgent.md           # 微博快讯
├── douyin-urgent.md          # 抖音短视频脚本
├── poster-urgent.md          # 海报文案
├── broadcast-script.md       # 广播稿
└── summary.md                # 内容包说明
```

### 步骤 5：审核与发布

应急内容同样需要审核，但走快速通道：

#### 快速审核流程
1. **AI初审**：敏感词、格式检查（2分钟）
2. **人工复核**：内容准确性确认（5分钟）
3. **紧急发布**：负责人确认后发布（3分钟）

**总用时目标**：10分钟内完成审核发布

#### 发布策略
- **红色预警**：全平台同步发布
- **橙色预警**：主要平台发布
- **黄色预警**：微博+微信发布
- **蓝色预警**：微博发布

### 步骤 6：效果追踪

发布后追踪效果：
- 发布后1小时阅读量
- 用户评论反馈
- 路况咨询电话变化
- 内容传播范围

记录到 `output/emergency/` 用于复盘。

## 应急类型详解

### 暴雨预警

#### 预警级别
- **蓝色**：12小时内降雨量将达50毫米以上
- **黄色**：6小时内降雨量将达50毫米以上
- **橙色**：3小时内降雨量将达50毫米以上
- **红色**：3小时内降雨量将达100毫米以上

#### 核心提示
- 降低车速，保持车距
- 开启雾灯、示廓灯
- 避免涉水行驶
- 注意低洼路段积水
- 遇积水绕行，不要强行通过

### 冰雪预警

#### 预警级别
- **蓝色**：48小时内最低气温下降8℃以上
- **黄色**：24小时内最低气温下降10℃以上
- **橙色**：24小时内最低气温下降12℃以上
- **红色**：24小时内最低气温下降16℃以上

#### 核心提示
- 安装防滑链
- 减速慢行，保持安全距离
- 避免急刹车、急转向
- 上坡提前加速，下坡低速挡
- 桥梁、隧道口易结冰，特别注意

### 大雾预警

#### 预警级别
- **黄色**：12小时内可能出现能见度<500米
- **橙色**：6小时内可能出现能见度<200米
- **红色**：2小时内可能出现能见度<50米

#### 核心提示
- 开启雾灯、近光灯、示廓灯、前后位灯
- 降低车速，保持车距
- 不要超车
- 必要时驶离高速
- 红色预警建议不要出行

### 台风预警

#### 预警级别
- **蓝色**：24小时内可能受热带气旋影响
- **黄色**：24小时内可能受热带气旋影响，平均风力8-9级
- **橙色**：12小时内可能受热带气旋影响，平均风力10-11级
- **红色**：6小时内可能受热带气旋影响，平均风力≥12级

#### 核心提示
- 减少不必要的外出
- 远离广告牌、大树
- 不要停在低洼地带
- 行车注意横风
- 台风天尽量不开车

## 约束规则

### 时效性要求
- **红色预警**：30分钟内完成内容生成
- **橙色预警**：45分钟内完成内容生成
- **黄色预警**：1小时内完成内容生成
- **蓝色预警**：2小时内完成内容生成

### 准确性要求
- 预警信息必须与官方一致
- 路况信息实时准确
- 不传播未经证实的信息
- 及时更新最新情况

### 发布规范
- 标明预警来源
- 注明发布时间
- 持续更新进展
- 预警解除后发布结束通知

### 安全提示原则
- 简明扼要，易于记忆
- 具体可操作
- 优先生命安全
- 避免制造恐慌

## 使用示例

### 示例 1：暴雨预警响应

```
用户: 市气象局发布了暴雨橙色预警，需要发应急内容

Agent 执行流程:
1. 识别类型：天气预警-暴雨-橙色
2. 读取模板：rainstorm.md
3. 收集信息：时间、区域、影响
4. 生成内容包：
   - 微信推文（完整版）
   - 微博快讯（精简版）
   - 抖音脚本（视频版）
   - 海报文案
5. 提交快速审核
6. 保存应急内容包
7. 提示发布注意事项
```

### 示例 2：道路管制响应

```
用户: 高速XX段发生事故，道路封闭，需要发布绕行提示

Agent 执行流程:
1. 识别类型：道路事件-事故-管制
2. 读取模板：road-closure.md
3. 收集信息：
   - 事故地点
   - 封闭路段
   - 预计时间
   - 绕行路线
4. 生成内容：
   - 路况通报
   - 绕行建议
   - 导航提示
5. 生成多平台版本
6. 输出完整内容包
```

### 示例 3：持续更新

```
用户: 暴雨预警升级了，从橙色升到红色，需要更新内容

Agent 执行流程:
1. 识别更新需求
2. 读取原内容
3. 更新预警级别
4. 强化安全提示
5. 增加紧急措施
6. 生成更新版本
7. 标注"更新"字样
8. 输出更新内容
```

```

Agent 执行流程:
1. 识别类型：预警解除
2. 生成解除通知
```
